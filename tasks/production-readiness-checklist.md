# 生产就绪检查清单

本清单用于判断 Tradgio 是否可以从本地 MVP 进入生产发布。任务完成记录以 `tasks/production-roadmap.md` 为准，本文件只记录发布门槛。

## 使用规则

- `[ ]` 表示未验证，`[x]` 表示已有可追溯证据。
- 每项通过时应在对应任务或验收记录中附测试结果、截图、CI 链接或演练记录。
- 标记为“阻断”的项目未通过时不得上线。
- 清单必须使用生产候选版本和测试环境执行，不能仅凭本地手动验证勾选。

## Gate 1：业务正确性

- [ ] 阻断：进货创建、增加、减少、删除明细和替换货品后的库存均正确。
- [ ] 阻断：出货创建、增加、减少、删除明细和替换货品后的库存均正确。
- [ ] 阻断：负库存只能由出货保存产生，并保留完整流水。
- [ ] 阻断：单据保存任一步骤失败后，单据、流水和快照均回到提交前状态。
- [ ] 阻断：重复提交不会重复影响库存。
- [ ] 阻断：不同账号无法读取、搜索、修改或导出对方数据。
- [ ] 阻断：单据编号不会因删除、缺号、并发或跨账号而重复。
- [ ] 报价单不影响库存。
- [ ] 货品和往来单位引用限制符合 PRD。
- [ ] 任务 26-31 已全部完成并有验收记录。

## Gate 3：本地架构与安全

- [x] 阻断：生产平台 ADR 已评审并锁定。
- [ ] 阻断：浏览器和仓库中不存在明文密码、可逆密码或备份密钥。
- [ ] 阻断：账号密码使用 PBKDF2-SHA-256、随机盐和版本化参数保存校验值。
- [ ] 阻断：Repository 与 File Adapter 强制当前账号隔离。
- [ ] 阻断：单据、库存流水和快照通过同一 IndexedDB transaction 提交。
- [ ] 阻断：编号具有账号内复合唯一索引约束。
- [ ] 合同附件 Blob 与元数据分离，并通过 File Adapter 访问。
- [ ] 页面层和领域层不直接调用 IndexedDB 或附件 Blob store。
- [ ] schema 版本化，升级失败不删除旧数据库且阻止继续写入。
- [ ] 日志不包含密码、派生密钥、备份密码、附件内容或不必要的业务敏感数据。

当前证据（尚不足以通过 Gate）：

- ADR 已锁定本地多账号、IndexedDB、Blob 附件、整机加密备份、PWA 和平台无关静态托管。
- 2026-06-12，任务 35 已通过 `specs/indexeddb-persistence.md` 和 `src/shared/persistence/indexeddbSchema.ts` 锁定账号隔离、复合唯一索引、单据库存事务、合同附件事务、schema 升级和错误契约；7 项契约测试通过。
- 2026-06-12，任务 37 已通过统一 Adapter 注册表锁定账号上下文、transaction、Blob、容量、加密、备份、迁移和错误映射；页面与领域模块浏览器 API 直连检查无结果，12 项契约测试通过。
- 上述条目当前仍是契约证据，安全密码和 IndexedDB/File Adapter 实现仍待任务 38-40 完成，因此 Gate 3 保持未通过。

## Gate 4：本地迁移与数据完整性

- [ ] 阻断：localStorage 到 IndexedDB 的迁移有版本号、幂等标记和完整性校验。
- [ ] 阻断：迁移失败保留旧数据，不留下半迁移状态。
- [ ] 阻断：迁移前后实体数量、金额、库存和附件核对一致。
- [ ] 明文密码只在成功登录后迁移，成功后删除对应明文。
- [ ] Base64 附件迁移到 Blob 后仍可下载，失败时保留旧附件。
- [ ] schema 升级兼容已有版本，版本过新时拒绝由旧程序写入。
- [ ] 已使用脱敏多账号样本完成至少两次迁移演练。

## Gate 6：备份、容量与恢复

- [ ] 阻断：错误密码、篡改、版本不兼容或容量不足时不修改现有数据。

- [ ] Storage API 展示容量；70% 提醒、85% 阻止高容量写入。
- [ ] 备份提醒、至少两处副本和恢复演练频率已记录。
- [ ] 用户已被明确告知忘记备份密码无法恢复。
