# 生产就绪检查清单

本清单用于判断 Tradgio 是否可以从本地 MVP 进入生产发布。任务完成记录以 `tasks/production-roadmap.md` 为准，本文件只记录发布门槛。

## 使用规则

- `[ ]` 表示未验证，`[x]` 表示已有可追溯证据。
- 每项通过时应在对应任务或验收记录中附测试结果、截图、CI 链接或演练记录。
- 标记为“阻断”的项目未通过时不得上线。
- 清单必须使用生产候选版本和测试环境执行，不能仅凭本地手动验证勾选。

## Gate 1：业务正确性

- [ ] 阻断：进货创建、增加、减少、删除明细和替换货品后的库存均正确。
- [ ] 阻断：出货创建、增加、减少、删除明细和替换货品后的库存均正确。
- [ ] 阻断：负库存只能由出货保存产生，并保留完整流水。
- [ ] 阻断：单据保存任一步骤失败后，单据、流水和快照均回到提交前状态。
- [ ] 阻断：重复提交不会重复影响库存。
- [ ] 阻断：不同账号无法读取、搜索、修改或导出对方数据。
- [ ] 阻断：单据编号不会因删除、缺号、并发或跨账号而重复。
- [ ] 报价单不影响库存。
- [ ] 货品和往来单位引用限制符合 PRD。
- [ ] 任务 26-31 已全部完成并有验收记录。

## Gate 2：自动化与工程质量

- [ ] 阻断：类型检查、单元测试、集成测试、E2E 和生产构建全部通过。
- [ ] 阻断：CI 会在失败时阻止合并或发布。
- [ ] 核心用户流程可在干净环境重复执行。
- [ ] 测试数据可独立创建和清理，不依赖开发者浏览器存量数据。
- [ ] lint 和格式检查通过。
- [ ] 依赖审计无未处置的高危或严重漏洞。
- [ ] 中危依赖风险已有升级、替换或接受记录。
- [ ] 关键失败保留可定位的日志、截图或追踪信息。

当前证据（尚不足以勾选 Gate）：
- 2026-06-10，任务 26 已建立 Vitest Node 测试基线；`npm run typecheck`、`npm run test`、`npm run build` 通过。
- 2026-06-11，当前完整自动化回归为 4 个测试文件、36 项全部通过；任务 26 记录的预期失败缺陷已在任务 27 关闭。
- 尚缺集成测试、E2E、CI、lint、格式检查和依赖风险处置，因此 Gate 2 保持未通过。

## Gate 3：生产架构与安全

- [ ] 阻断：生产平台 ADR 已评审并锁定。
- [ ] 阻断：浏览器和仓库中不存在明文密码、服务端密钥或长期有效敏感令牌。
- [ ] 阻断：服务端强制账号隔离，不能只依赖客户端过滤。
- [ ] 阻断：单据与库存通过数据库事务提交。
- [ ] 阻断：编号具有数据库唯一约束。
- [ ] 合同附件二进制存入对象存储，数据库仅保存元数据和存储键。
- [ ] 附件下载经过身份与归属校验。
- [ ] 页面层和领域层不直接依赖供应商 SDK。
- [ ] 环境变量按开发、测试、生产隔离，`.env.example` 已提供非敏感字段清单。
- [ ] 错误日志不包含密码、令牌、附件内容或不必要的业务敏感数据。

## Gate 4：数据迁移

- [ ] 阻断：本地数据导出格式有版本号和完整性校验。
- [ ] 阻断：迁移工具支持预检查、幂等导入、失败重试和结果报告。
- [ ] 阻断：迁移前后实体数量、金额、库存和附件核对一致。
- [ ] 同一数据包重复执行不会生成重复数据或重复库存影响。
- [ ] 无法迁移的记录会被明确列出，不会静默丢弃。
- [ ] 已使用脱敏样本完成至少两次测试环境迁移演练。
- [ ] 正式迁移窗口、负责人和回滚条件已确认。

## Gate 5：部署与运行

- [ ] 阻断：测试环境已从零成功部署。
- [ ] 阻断：生产发布流程包含质量门禁、数据库迁移和人工确认。
- [ ] 阻断：已成功演练恢复上一稳定版本。
- [ ] 健康检查能识别前端、API、数据库和对象存储异常。
- [ ] 静态资源和模板文件在生产环境可访问。
- [ ] 导出依赖不会阻塞首屏加载，构建产物体积已有评估。
- [ ] 运行文档包含环境变量、部署、迁移、回滚和常见故障处理。

## Gate 6：监控、备份与恢复

- [ ] 阻断：关键前端和服务端错误可被监控并触发告警。
- [ ] 阻断：数据库与对象存储均配置备份。
- [ ] 阻断：已从备份恢复数据并通过核心业务校验。
- [ ] 告警联系人、严重级别和响应方式已明确。
- [ ] 备份保留周期、恢复目标和演练频率已记录。
- [ ] 外部 Auth、数据库、存储或导出服务异常时有明确用户反馈。

## Gate 7：UAT 与发布结论

- [ ] 阻断：真实业务样本完成货品、单位、进货、出货、报价和合同 UAT。
- [ ] 阻断：出货单与报价单模板 Excel 内容和样式经用户确认。
- [ ] 阻断：无未关闭的 P0/P1 缺陷。
- [ ] 主流桌面浏览器完成关键流程验证。
- [ ] 用户确认迁移结果和上线窗口。
- [ ] 上线版本、提交号、数据库迁移版本和回滚版本已记录。
- [ ] 任务 46 已形成最终“允许上线”或“暂缓上线”结论。
