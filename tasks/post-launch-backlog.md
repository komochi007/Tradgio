# 上线后需求池

本文档用于记录 Tradgio 正式上线后的 Bug、体验优化、新功能和运维事项。上线前生产化任务已在 `tasks/production-roadmap.md` 完成；后续日常迭代不再追加到生产化路线图。

管理规则见 `docs/release-management.md`。发布检查见 `tasks/release-checklist.md`。发布历史见 `tasks/release-log.md`。

## 1. 使用规则

- 新反馈先登记，再分级和排期。
- 信息不完整时状态保持 `待澄清`，不要直接进入开发。
- 每次补丁版只选择少量高确定性的事项，优先降低正式使用风险。
- 修改超过 3 个文件时，先拆成小步骤。
- 涉及 IndexedDB schema、备份恢复、账号隔离、库存、导出模板、PWA 更新的事项，风险等级必须标为 `高`。

## 2. 状态与优先级

状态：`待澄清`、`待排期`、`进行中`、`待验收`、`已发布`、`暂缓`、`关闭`。

优先级：

- `P0`：数据丢失、无法登录、核心录单不可用、备份恢复不可用、安全问题，立即热修。
- `P1`：核心业务错误但有绕行办法，进入最近补丁版。
- `P2`：明确提升效率的小功能或体验优化，排入后续补丁/小版本。
- `P3`：非紧急优化、想法池、低频需求，暂缓评估。

风险等级：

- `低`：文案、样式、无业务逻辑变化。
- `中`：表单、列表、路由、搜索、导出触发入口等用户可见流程。
- `高`：库存、账号隔离、IndexedDB、附件、备份恢复、schema、导出模板、Service Worker。

### PL-002：出货单订单号与模板 Excel 导出修复

- 类型：Bug / 优化
- 优先级：P1
- 状态：已发布
- 风险等级：中
- 目标版本：`0.1.1`
- 所属模块：出货 / 导出
- 登记日期：2026-07-02
- 来源：用户反馈

问题或目标：

- 出货单模板 Excel 在明细行超过模板基础行数后，底部合并单元格内容出现重复显示。
- 出货单模板中的“订单号”需要来自新建出货单时手动输入的业务订单号，不能直接使用系统单据编号。
- 导出模板表格区域文字需要保持单行显示，不自动换行。
- 出货单导出时，产品编号、尺寸、颜色等可选字段整列为空时，模板中不保留对应表头列；备注列始终保留。

影响用户：

- 使用出货单模板 Excel 交付客户或打印签收的业务用户。

验收标准：

- 新建 / 编辑出货单必须填写订单号，且订单号与系统单据编号分开展示和保存。
- 出货单模板 Excel 的 `送货单 NO:` 与“订单号”使用手动订单号；历史无订单号单据导出时对应位置留空。
- 明细行扩展后，合计、送货人、收货人和说明行不再重复显示。
- 表格区域文字不自动换行。
- 产品编号、尺寸、颜色整列为空时隐藏对应列；备注列即使为空也保留。

实施拆分：

- 出货单数据模型、表单、详情、列表和搜索接入 `customerOrderNo`。
- 导出 payload 增加手动订单号，并保持系统单据编号用于内部编号和下载文件名。
- 修复出货模板 Excel 动态行、合并区域、换行和空列隐藏逻辑。
- 补充单元测试、E2E 流程数据和发布管理文档。

验证命令：

- `npm run test -- src/modules/export-service/exportAdapter.test.ts src/modules/document-core/documentTransaction.test.ts`
- `npm run quality`
- `npm run test:e2e`

发布记录：

- 已随 `0.1.1` 发布，发布提交号 `9b377ed4fbaea82020029b2594b7a73eca66a43e`，详见 `tasks/release-log.md`。
