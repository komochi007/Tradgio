# 开发任务总表

本文件用于维护项目的稳定开发任务拆解。

使用规则：
- 任务按优先级排序
- 每个任务控制在一次对话可完成的粒度
- 依赖关系必须明确
- 模块归属以 `docs/architecture.md` 为准
- 若任务边界发生变化，应同步更新本文件

编号规则：
- 全局开发路线使用连续数字编号，例如 `任务 01`、`任务 26`。
- 专项优化使用阶段前缀编号；第三轮优化为 `OPT3-01` 至 `OPT3-09`，未来专项可使用 `OPT4-*`。
- 后续生产化全局任务从 `任务 26` 开始，不占用专项优化编号。
- Git 历史不重写，旧提交中的“本轮优化任务 1-9”通过 `docs/optimization-plan.md` 的旧称映射追溯。

---

### 任务 01：工程目录与模块骨架重组
- 描述：按 `architecture.md` 的模块边界把当前初始化项目整理为可持续扩展的结构，至少区分 `App Shell`、`Shared Platform`、`Auth`、`Overview`、`Master Data`、`Document Core`、`Inventory Engine`、`Contract Center`、`Search`、`Export Service` 的代码入口与占位目录。
- 所属模块：`App Shell` / `Shared Platform`
- 验收标准：代码目录已按模块和分层拆开；路由入口、页面入口、共享样式与基础工具位置稳定；后续功能不需要再改一次大结构。
- 依赖：无
- 预计对话轮数：2-3 轮

### 任务 02：Shared Platform 基础能力初始化
- 描述：补齐 `Shared Platform` 的最小基础设施，包括环境配置读取、Query Client、错误映射、通知入口、通用格式化与 validator 占位接口。
- 所属模块：`Shared Platform`
- 验收标准：全局 provider 已接入；共享工具有统一出口；后续模块可以通过共享层接入数据、校验和通知，而不是各自散落实现。
- 依赖：任务 01
- 预计对话轮数：2-3 轮

### 任务 03：Auth 适配层与登录态模型
- 描述：定义 `Auth` 模块的接口、状态模型和适配层边界，先用可替换的 mock / local 实现打通 `register`、`login`、`logout`、`restoreSession`、`getCurrentAccount`。
- 所属模块：`Auth`
- 验收标准：存在统一 Auth 接口；登录态可恢复；未登录与已登录状态可被页面消费；后续可替换为真实托管 Auth。
- 依赖：任务 02
- 预计对话轮数：2-3 轮

### 任务 04：登录页、注册页与受保护路由
- 描述：落地 `/login`、`/register`、登录拦截、会话恢复和未登录跳转逻辑，把当前 `App Shell` 改造成登录后区域。
- 所属模块：`App Shell` / `Auth`
- 验收标准：未登录访问业务路由会跳到登录页；登录成功进入总览页；刷新页面后可恢复会话；退出登录后回到登录页。
- 依赖：任务 03
- 预计对话轮数：2-3 轮

### 任务 05：基础组件层第一批落地
- 描述：按 `frontend-spec.md` 和 `design.md` 落地第一批基础组件：`Button`、`Input`、`Select`、`Tag`、`EmptyState`、`Skeleton`、`SectionCard`。
- 所属模块：`Shared Platform`
- 验收标准：这些组件已从页面中抽离；支持文档要求的基础状态；后续业务页不再直接手写同类控件样式。
- 依赖：任务 01
- 预计对话轮数：2-3 轮

### 任务 06：Master Data - 货品管理页面
- 描述：实现货品列表页和新建/编辑页，包含搜索、空状态、表单校验、状态切换与本地数据持久化入口。
- 所属模块：`Master Data`
- 验收标准：可新增、编辑、查看、停用货品；列表与表单具备加载 / 空 / 错误 / 正常状态；货品数据可被后续录单流程引用。
- 依赖：任务 02、任务 05
- 预计对话轮数：2-3 轮

### 任务 07：Master Data - 往来单位管理页面
- 描述：实现客户 / 供应商列表页和新建/编辑页，包含类型切换、搜索、表单校验与引用能力。
- 所属模块：`Master Data`
- 验收标准：可新增、编辑、查看、停用客户与供应商；录单场景可区分客户和供应商来源；页面状态完整。
- 依赖：任务 02、任务 05
- 预计对话轮数：2-3 轮

### 任务 08：Inventory Engine 数据模型与库存计算内核
- 描述：落地 `InventoryLedger`、`CurrentStockSnapshot` 的前端领域模型与计算函数，明确 `applyPurchaseOrder`、`applySalesOrder`、`recalculateOrderDelta` 的输入输出。
- 所属模块：`Inventory Engine`
- 验收标准：库存增减与改单差额回算逻辑可独立调用；没有页面直接写库存；负库存规则和库存历史模型已明确。
- 依赖：任务 02
- 预计对话轮数：2-3 轮

### 任务 09：Document Core - 进货单创建与列表
- 描述：实现进货单列表页、新建页、详情页基础版，接入供应商与货品选择、明细行增删、金额计算、整单保存。
- 所属模块：`Document Core`
- 验收标准：可以创建进货单；至少一条明细必填；保存后生成单据记录；页面状态与表单校验完整。
- 依赖：任务 06、任务 07、任务 08
- 预计对话轮数：2-3 轮

### 任务 10：Document Core + Inventory Engine - 进货库存联动与改单回算
- 描述：把进货单保存与 `Inventory Engine` 联动，并补齐进货单编辑后的库存差额回算。
- 所属模块：`Document Core` / `Inventory Engine`
- 验收标准：新建进货单后库存增加；修改已保存进货单后库存按差额回算；不能绕过库存引擎直接修改库存快照。
- 依赖：任务 09
- 预计对话轮数：2-3 轮

### 任务 11：Document Core - 出货单创建与库存不足警告
- 描述：实现出货单列表页、新建页、详情页基础版，接入客户与货品选择、明细行编辑、当前库存展示与库存不足警告。
- 所属模块：`Document Core`
- 验收标准：可以创建出货单；录入数量大于当前库存时会出现明确警告；用户确认后仍允许保存。
- 依赖：任务 06、任务 07、任务 08
- 预计对话轮数：2-3 轮

### 任务 12：Document Core + Inventory Engine - 出货库存联动与改单回算
- 描述：把出货单保存与 `Inventory Engine` 联动，并补齐出货单编辑后的库存差额回算与负库存保留规则。
- 所属模块：`Document Core` / `Inventory Engine`
- 验收标准：新建出货单后库存减少；修改已保存出货单后库存按差额回算；负库存仅来自出货保存且有流水记录。
- 依赖：任务 11
- 预计对话轮数：2-3 轮

### 任务 13：Document Core - 报价单创建与详情
- 描述：实现报价单列表页、新建页、详情页，支持货品引用、默认销售价带出、手动改价、金额汇总。
- 所属模块：`Document Core`
- 验收标准：可以创建、编辑、查看报价单；默认销售价可带出；报价单保存不影响库存。
- 依赖：任务 06、任务 07
- 预计对话轮数：2-3 轮

### 任务 14：Overview 总览页聚合
- 描述：把当前占位总览页替换为真实聚合页，展示最近记录、快捷操作、库存摘要和后续提醒区。
- 所属模块：`Overview`
- 验收标准：总览页能读取最近进货 / 出货 / 报价记录和库存摘要；无数据时有明确空状态；快捷入口可跳到新建流程。
- 依赖：任务 10、任务 12、任务 13
- 预计对话轮数：2-3 轮

### 任务 15：Contract Center - 合同列表、上传、详情
- 描述：实现合同列表页、上传页、详情页，包含合同元数据表单、附件选择、附件元数据记录与按客户筛选。
- 所属模块：`Contract Center`
- 验收标准：可新增合同记录；附件与元数据分离处理；上传成功后可在详情页查看附件信息；页面状态完整。
- 依赖：任务 02、任务 07、任务 05
- 预计对话轮数：2-3 轮

### 任务 16：Search 统一搜索结果模型与查询页
- 描述：实现统一搜索结果结构、搜索输入区、结果列表和类型标签，聚合进货、出货、报价、合同四类记录。
- 所属模块：`Search`
- 验收标准：查询页返回统一结果结构；支持按货品、客户、供应商、单据编号搜索；搜索前、搜索中、无结果、异常状态清晰区分。
- 依赖：任务 10、任务 12、任务 13、任务 15
- 预计对话轮数：2-3 轮

### 任务 17：Export Service 接口边界与导出占位实现
- 描述：先实现 `buildExportPayload` 和 `Export Service` 调用边界，用 mock / placeholder 方式打通导出入口，不提前绑定真实模板细节。
- 所属模块：`Export Service` / `Document Core`
- 验收标准：进货、出货、报价单详情页都有统一导出入口；导出 payload 结构统一；导出失败不影响已保存数据。
- 依赖：任务 09、任务 11、任务 13
- 预计对话轮数：2-3 轮

### 任务 18：全局状态完善与异常反馈收口
- 描述：补齐骨架屏、空状态、字段级错误、页面级错误、提交中状态、通知反馈等全局交互规范。
- 所属模块：`Shared Platform` / `App Shell`
- 验收标准：核心页面都具备加载 / 空 / 错误 / 正常状态；表单错误就近展示；按钮提交中不抖动布局；全局提示风格统一。
- 依赖：任务 05、任务 14、任务 15、任务 16
- 预计对话轮数：2-3 轮

### 任务 19：本地数据持久化与替换真实后端的适配点整理
- 描述：整理当前 mock / local 持久化方案的统一出口，明确未来替换托管 Auth、PostgreSQL、Object Storage、Serverless Export 的 adapter 接入点。
- 所属模块：`Shared Platform`
- 验收标准：当前开发环境下数据链路稳定；适配器边界清晰；后续替换真实平台能力时不需要重写页面层和领域层。
- 依赖：任务 03、任务 10、任务 12、任务 15、任务 17
- 预计对话轮数：2-3 轮

### 任务 20：MVP 验证用例与交付自检
- 描述：按 `architecture.md` 的 MVP 验证场景建立一套最小自检清单，覆盖登录、基础资料、单据、库存、合同、搜索、导出。
- 所属模块：`Shared Platform`
- 验收标准：存在可执行的自检路径；至少覆盖架构文档列出的关键验证场景；后续每完成一批功能都能快速回归。
- 依赖：任务 04、任务 10、任务 12、任务 13、任务 15、任务 16、任务 17
- 预计对话轮数：1-2 轮

## 第二阶段优化任务

### 任务 21：App Shell 视觉重构与导航收紧
- 描述：重构 `App Shell` 的品牌区、左侧导航、顶部页头和底部帐号卡片，落地新产品图标、导航图标、激活态、帐号卡片和退出登录位置调整。
- 所属模块：`App Shell` / `Auth`
- 验收标准：导航仅保留图标和标题；激活态为主蓝底白字且无边框；顶部不再展示账号与退出按钮；底部帐号卡片显示用户名与退出按钮；除总览页外不再使用顶部大卡片。
- 依赖：任务 05、任务 18
- 预计对话轮数：2-3 轮

### 任务 22：Overview 重做与查询并入
- 描述：重做总览页欢迎卡片、快捷入口图标、Bento 区卡片比例，并把跨模块搜索从独立页面迁移为总览页内联搜索卡片与结果区；天气信息采用定位城市，失败回退上海。
- 所属模块：`Overview` / `Search`
- 验收标准：总览页展示英文欢迎语、日期和天气；库存概览与最近单据卡片尺寸对齐；搜索功能在总览页可直接完成查询与跳转；左侧导航不再出现“查询”入口。
- 依赖：任务 14、任务 16、任务 21
- 预计对话轮数：3-4 轮

### 任务 23：表单控件与搜索交互统一升级
- 描述：统一全局下拉框样式；所有列表页改为点击“搜索”按钮后才触发搜索；合同列表页移除客户筛选；为进货、出货、报价录单页引入可搜索货品选择组件。
- 所属模块：`Shared Platform` / `Master Data` / `Document Core` / `Contract Center`
- 验收标准：下拉框均为圆角白底、选中项为圆角浅灰底；搜索不再输入即过滤；合同页不再显示客户筛选；录单页货品字段支持输入搜索并正确回填关联字段。
- 依赖：任务 05、任务 09、任务 11、任务 13、任务 15、任务 21
- 预计对话轮数：3-4 轮

### 任务 24：货品字段扩展与单据编号规则调整
- 描述：为货品增加 `产品类型` 字段，收敛单位选项，并统一调整进货、出货、报价、合同的编号生成规则为月份后两位流水号 `01-99`。
- 所属模块：`Master Data` / `Document Core` / `Contract Center`
- 验收标准：货品新增 / 编辑 / 展示链路均支持 `产品类型`；单位仅保留 `个 / 件 / 箱 / 卷 / 码`；新生成编号符合两位流水规则；不破坏现有数据读取。
- 依赖：任务 06、任务 09、任务 11、任务 13、任务 15
- 预计对话轮数：2-3 轮

### 任务 25：列表行导出入口补齐与第二阶段回归验收
- 描述：在进货、出货、报价列表页的每行操作列增加导出按钮与格式下拉，并对第二阶段所有优化项执行构建、自动测试和手动回归。
- 所属模块：`Export Service` / `Document Core` / `Shared Platform`
- 验收标准：三类单据列表页均有行级导出入口；导出格式下拉符合统一样式；导出复用现有 `Export Service`；第二阶段优化项具备明确回归记录。
- 依赖：任务 17、任务 22、任务 23、任务 24
- 预计对话轮数：2-3 轮
