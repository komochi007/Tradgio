# Tradgio 设计规范

## 1. 文档定位

本文档用于定义 Tradgio 在 MVP 阶段的整体视觉风格、设计 token、排版规则、布局规则、组件微规范与交互动效原则，作为后续 UI 设计、前端开发和设计系统沉淀的统一依据。

适用范围：
- Web 桌面端主应用
- 总览页、列表页、详情页、录单页、登录注册页

当前阶段不包含：
- 暗黑模式正式落地
- 手机端深度适配
- 多品牌或多主题体系

## 2. 产品气质定义

### 2.1 核心关键词

- 精致
- 冷静
- 高效

### 2.2 风格结论

Tradgio 是一个面向国内贸易个体经营者和小公司老板的单人高频业务工具。视觉风格不应走重装饰、强科技感或营销化路线，而应强调稳定、专业、低疲劳、可连续使用。

整体采用：
- `Bento Grid` 的分块秩序感
- 大厂 `B 端` 的清晰结构和专业控制感
- 浅色基调下的冷灰中性色体系
- 蓝色作为少量关键强调色

设计落点：
- 总览页强调结构感和模块分区，可使用 Bento 风格
- 列表页、表单页、录单页优先保证效率，不做过强装饰
- 全站不追求“炫”，追求“稳、准、清楚”

### 2.3 场景定义

典型使用场景为：用户在白天或常规室内光线环境下，长时间坐在电脑前进行录单、查单、找货品、找客户、导出单据等连续操作。

因此界面必须满足：
- 长时间观看不疲劳
- 数据与表单信息一眼可扫
- 主次操作明显，但不过度刺激
- 空状态、错误状态、加载状态都足够直接

## 3. 主题策略

### 3.1 当前策略

- 本期仅落地浅色主题
- 设计 token 采用语义化命名，预留后续暗黑模式扩展能力

### 3.2 主题约束

- 组件内不直接写死业务颜色
- 优先使用语义 token，如 `text-primary`、`surface-base`、`border-default`
- 后续若支持暗黑模式，仅通过 token 覆盖完成，不重写组件结构

## 4. 色彩系统

### 4.1 用色原则

- 中性色为主，蓝色为辅
- 页面主体维持白、浅灰、冷灰蓝关系
- 蓝色只用于主操作、选中态、焦点态和关键信息强调
- 状态色仅用于反馈，不参与品牌表达
- 不使用纯黑 `#000000` 和纯白 `#FFFFFF` 作为大面积背景对比关系的唯一手段

### 4.2 核心色板

| 角色 | Token | Hex | 用途 |
|---|---|---|---|
| 主色 | `color-primary` | `#2563EB` | 主按钮、链接、选中、焦点高亮 |
| 主色悬停 | `color-primary-hover` | `#1D4ED8` | 主按钮 Hover |
| 主色按下 | `color-primary-active` | `#1E40AF` | 主按钮 Active |
| 辅助蓝 | `color-accent` | `#93C5FD` | 轻强调、图表辅助、高亮背景 |
| 画布背景 | `bg-canvas` | `#F6F8FB` | 应用整体背景 |
| 一级表面 | `surface-base` | `#FFFFFF` | 卡片、弹层、输入框背景 |
| 二级表面 | `surface-subtle` | `#F8FAFC` | 次级分区、弱容器背景 |
| 边框默认 | `border-default` | `#E5E7EB` | 容器、输入框、分割线 |
| 文本一级 | `text-primary` | `#111827` | 主标题、正文、关键数据 |
| 文本二级 | `text-secondary` | `#4B5563` | 次级说明、列表辅助信息 |
| 文本三级 | `text-tertiary` | `#9CA3AF` | 占位、弱标签、禁用辅助文本 |

### 4.3 状态色

| 角色 | Token | Hex | 用途 |
|---|---|---|---|
| 成功 | `state-success` | `#16A34A` | 成功提示、已完成状态 |
| 警告 | `state-warning` | `#D97706` | 风险提醒、库存不足提示 |
| 错误 | `state-error` | `#DC2626` | 表单错误、失败提示 |
| 信息 | `state-info` | `#2563EB` | 系统说明、信息提示 |

### 4.4 色彩使用比例

- 中性色占比建议 `85%` 以上
- 主蓝色占比建议 `10%` 以内
- 状态色只在必要反馈中局部出现

### 4.5 扩展建议

图表如需扩展颜色，仍以低饱和冷色为主，避免引入大面积高饱和多色，防止与整体风格冲突。

## 5. 字体与排版

### 5.1 字体族

推荐字体栈：

```css
"PingFang SC", "Noto Sans SC", "Microsoft YaHei", sans-serif
```

设计原则：
- 中文优先可读性
- 不引入装饰型字体
- 保持清洁、克制、现代

数字和单据编号建议：
- 使用等宽数字特性，增强金额和编号的纵向对齐感

### 5.2 字重建议

| 层级 | 字重 |
|---|---|
| 页面主标题 | 600 |
| 区块标题 | 600 |
| 表头 / 强调文本 | 500 / 600 |
| 正文 | 400 |
| 辅助说明 | 400 |

### 5.3 字号与行高层级

| 层级 | 字号 | 行高 | 用途 |
|---|---:|---:|---|
| H1 | 32px | 40px | 页面主标题、登录页主标题 |
| H2 | 24px | 32px | 页面区块标题、大模块标题 |
| H3 | 18px | 28px | 卡片标题、表单分组标题 |
| Body | 14px | 22px | 正文、表格、表单、常规说明 |
| Caption | 12px | 18px | 辅助文案、标签、占位信息 |

### 5.4 排版规则

- 业务页面以 `14px` 为主，不默认使用 `16px` 正文
- 标题层级通过字号与字重同时拉开
- 一屏内避免出现过多字号
- 文本对齐优先左对齐
- 大段说明文字控制在可快速扫读的长度内

## 6. 布局与栅格

### 6.1 设计基准

- 优先桌面端
- 兼容平板
- 移动端只保证基础可用

### 6.2 响应式断点

| 设备 | 断点 |
|---|---|
| Mobile | `< 768px` |
| Tablet | `768px - 1279px` |
| Desktop | `>= 1280px` |

### 6.3 栅格体系

- 采用 `8px` 网格体系
- 允许 `4px` 作为细部微调单位

基础间距 token：

| Token | 数值 |
|---|---:|
| `space-1` | 4px |
| `space-2` | 8px |
| `space-3` | 12px |
| `space-4` | 16px |
| `space-5` | 20px |
| `space-6` | 24px |
| `space-8` | 32px |
| `space-10` | 40px |
| `space-12` | 48px |

### 6.4 页面容器

主应用结构：
- 左侧固定导航
- 右侧内容区域自适应

内容区规范：
- `max-width: 1440px`
- Desktop 内边距：`32px`
- Tablet 内边距：`24px`
- Mobile 内边距：`16px`

### 6.5 页面布局原则

- 总览页使用 `12` 栏布局，可做异构卡片
- 列表页以筛选区、工具栏、表格区为主，不追求拼贴感
- 详情页和录单页采用分区式布局，强调信息分组和操作顺序

## 7. 圆角、边框与阴影

### 7.1 圆角体系

整体风格偏柔和，圆角相对偏大，但仍保持专业感。

| 场景 | 数值 |
|---|---|
| 小型元素 | 10px |
| 输入框 / 按钮 | 14px |
| 中型卡片 | 20px |
| 大型面板 | 24px |

### 7.2 边框原则

- 默认使用 `1px` 浅边框
- 通过边框和浅背景划分层级，不依赖重阴影
- 不使用夸张的高对比描边

### 7.3 阴影体系

默认阴影：

```css
0 1px 2px rgba(16, 24, 40, 0.04), 0 8px 24px rgba(16, 24, 40, 0.04)
```

Hover 阴影：

```css
0 4px 12px rgba(16, 24, 40, 0.06), 0 16px 32px rgba(16, 24, 40, 0.06)
```

使用原则：
- 阴影只作为层级补充
- 数据密集页面避免过度悬浮感
- 不做玻璃拟态风格

## 8. 组件微规范

### 8.1 按钮

#### 主按钮

- 背景：`#2563EB`
- 文本：`#FFFFFF`
- 高度：`40px`
- 横向内边距：`16px`
- 圆角：`14px`

状态规则：
- Hover：背景 `#1D4ED8`
- Active：背景 `#1E40AF`
- Disabled：背景 `#D1D5DB`，文本 `#F9FAFB`
- Focus：`0 0 0 4px rgba(37, 99, 235, 0.18)`

#### 次要按钮

- 背景：`#FFFFFF`
- 文本：`#111827`
- 边框：`1px solid #D1D5DB`
- 高度：`40px`
- 圆角：`14px`

状态规则：
- Hover：背景 `#F8FAFC`
- Active：背景 `#F1F5F9`
- Disabled：文本和边框统一弱化

#### 幽灵按钮

- 背景：透明
- 文本：`#4B5563`
- 圆角：`12px`

状态规则：
- Hover：背景 `#EFF6FF`
- Active：背景 `#DBEAFE`
- Focus：使用轻蓝外环

#### 按钮尺寸

| 尺寸 | 高度 |
|---|---|
| Small | 36px |
| Default | 40px |
| Large | 44px |

### 8.2 输入框

#### 默认状态

- 高度：`40px`
- 背景：`#FFFFFF`
- 边框：`1px solid #D1D5DB`
- 圆角：`14px`
- 文本：`#111827`

#### Focus 状态

- 边框：`#2563EB`
- 阴影：`0 0 0 4px rgba(37, 99, 235, 0.14)`

#### 错误状态

- 边框：`#DC2626`
- 阴影：`0 0 0 4px rgba(220, 38, 38, 0.12)`
- 错误文案使用 `12px`

#### Disabled 状态

- 背景：`#F3F4F6`
- 文本：`#9CA3AF`

输入规则：
- 表单控件视觉高度统一
- 明细录入区域避免混用多种输入样式
- 占位文字只承担辅助作用，不替代标签

### 8.3 卡片

卡片主要用于：
- 总览页模块承载
- 详情页结构分区
- 登录注册页局部容器

规范：
- 圆角：`20px`
- 背景：`#FFFFFF`
- 边框：`1px solid #E5E7EB`
- 默认阴影使用全局浅阴影体系

原则：
- 列表页减少卡片嵌套
- 不做“卡片套卡片套卡片”
- 高密度信息优先分区，不优先堆阴影容器

## 9. 页面级应用规则

### 9.1 应用框架

- 一级导航采用左侧栏
- 顶部可保留页面标题、筛选操作和全局操作区域
- 导航、内容、辅助操作区之间层级清晰分离

### 9.2 总览页

总览页是全站最适合体现 `Bento Grid` 风格的页面。

建议结构：
- 顶部欢迎信息
- 快捷操作区
- 最近单据区
- 关键业务摘要区
- 合同或查询入口区

视觉原则：
- 可使用异构卡片布局
- 可使用轻微的浅色分区背景
- 强调“今天先做什么”和“最近发生了什么”

### 9.3 列表页

适用页面：
- 货品
- 往来单位
- 进货
- 出货
- 报价
- 合同
- 查询

视觉原则：
- 先效率，后装饰
- 筛选栏、工具栏、表格主体分层明确
- 单屏尽可能容纳更多有效记录
- 操作按钮与批量操作关系清楚

### 9.4 录单页与详情页

建议结构：
- 主信息区
- 明细列表区
- 汇总与操作区

原则：
- 输入路径顺序自然
- 明细表区域重点突出
- 保存、导出、返回等操作保持稳定位置
- 错误反馈就近显示，不打断主流程

### 9.5 登录与注册页

风格上比业务页更干净，但不额外做重品牌化包装。

建议表现：
- 居中主卡片
- 简洁标题
- 稳定输入框和按钮节奏
- 可加入非常轻的背景层次，不做复杂插画

## 10. 交互与动效

### 10.1 全局过渡规范

默认过渡：

```css
180ms cubic-bezier(0.22, 1, 0.36, 1)
```

较重元素过渡：

```css
240ms cubic-bezier(0.22, 1, 0.36, 1)
```

### 10.2 动效原则

- 优先动画 `opacity`、`transform`、`box-shadow`
- 不做弹跳、橡皮筋、强烈缩放效果
- 不通过动画制造多余存在感
- 动效的目的应始终是帮助用户理解状态变化

### 10.3 Loading 原则

- 页级加载优先用骨架屏
- 骨架结构尽量贴近真实布局
- 按钮提交中只替换按钮内部状态，不导致布局抖动
- 表格刷新优先局部加载，不锁死整页

### 10.4 Empty State 原则

- 空状态必须包含明确下一步动作
- 文字短、直白、业务化
- 至少提供一个主操作入口

推荐结构：
- 标题
- 一句说明
- 主按钮

### 10.5 错误反馈原则

- 错误信息尽量就近出现
- 表单错误优先字段级提示
- 页面级错误提供刷新或返回方案
- 避免大面积红色污染整个界面

## 11. 文案与信息表达

### 11.1 文案风格

- 直接
- 清楚
- 业务导向
- 不说空话

### 11.2 文案原则

- 按动作写，不按概念写
- 优先告诉用户下一步该做什么
- 成功、失败、空状态文案尽量短
- 避免过度拟人化或情绪化表达

示例方向：
- “还没有货品，先新建一个货品”
- “库存不足，请确认是否继续保存”
- “页面加载失败，请刷新后重试”

## 12. 可访问性与可用性要求

- 文本与背景保证足够对比度
- Focus 状态必须清晰可见
- 交互控件高度不低于 `36px`
- 不仅依赖颜色传递状态，必要时配合图标或文本
- 表格、表单、按钮在键盘导航下应有清晰焦点顺序

## 13. 设计落地优先级

### P0

- 色彩 token
- 字体与排版层级
- 左侧导航应用框架
- 按钮、输入框、卡片规范
- 总览页 / 列表页 / 录单页三类页面骨架

### P1

- 状态标签与提示反馈体系
- 表格样式统一
- 弹层、抽屉、下拉选择器规范

### P2

- 暗黑模式扩展
- 图表色板扩展
- 更细致的插图或空状态视觉资产

## 14. 最终设计结论

Tradgio 的视觉系统应建立在“冷静中性色 + 精准蓝色强调 + 柔和大圆角 + 高效率信息密度”的组合上，用总览页承接品牌感，用列表和录单页承接业务效率。

它不是面向团队协作的复杂后台，也不是强调展示感的 SaaS 官网，而是一套服务于个体经营者日常高频工作的、稳定且高级的桌面业务界面系统。
