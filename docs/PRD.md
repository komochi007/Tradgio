### 1. 需求背景

- **目标用户现状**：目标用户是国内贸易个体经营者和小公司老板，通常只有 1 个人高频使用系统，自己负责货品管理、进货、出货、报价、合同保存和日常单据处理。
- **核心痛点**：现在主要依赖表格分散记录，业务信息分布在多个文件里，查找慢、容易漏，单据导出不统一，合同资料不好集中保存和快速定位。
- **当前问题汇总**：

| 问题点 | 问题描述 | 当前影响 |
|--------|----------|----------|
| 记录分散 | 货品、进货、出货、报价、合同分别记录在不同表格或文件中 | 日常操作来回切换，容易漏记 |
| 查询缓慢 | 想找某个货、某个客户或某张单据时，需要翻多个表格 | 查找效率低，影响日常跟单 |
| 单据不统一 | 进货单、出货单、报价单需要按固定格式整理 | 重复手工整理，容易出错 |
| 合同不好管 | 合同文件和合同信息没有集中整理 | 后续查合同、找客户资料不方便 |

### 2. 需求目标

- **业务目标**：
  - 让用户在一个平台内完成货品、往来单位、进货、出货、报价、合同的日常管理
  - 让用户可以快速查到某个货品、客户、供应商、单据编号对应的业务记录
  - 让用户稳定导出符合既有格式的进货单、出货单、报价单

- **成功标准**：

| 指标 | 当前状态 | 目标状态 |
|------|----------|----------|
| 日常记录方式 | 依赖多个表格和文件 | 在一个平台内完成主要记录 |
| 查找效率 | 需要翻多个文件 | 可通过搜索快速定位结果 |
| 单据整理 | 需要手工套格式 | 可直接导出既有格式单据 |
| 历史延续性 | 换电脑后整理麻烦 | 登录后可继续查看历史记录 |
| 使用频率 | 可能回退到原有表格流程 | 能持续作为日常高频工具使用 |

### 3. 需求范围

- **包含**：
  - 注册与登录
  - 总览页
  - 货品管理
  - 往来单位管理
  - 进货单创建、编辑、详情、导出
  - 出货单创建、编辑、详情、导出
  - 报价单创建、编辑、详情、导出
  - 合同上传、查看、查询
  - 综合查询

- **不包含**：
  - 多人协作和复杂权限管理
  - 复杂财务对账、收付款跟踪、利润核算
  - 库存预警和自动提醒
  - 多套单据模板管理
  - 平台内直接生成合同正文
  - 手机端优先体验

## 四、功能说明

### 功能 1：注册与登录

**功能描述**

用户可在注册页创建账号，在登录页输入账号和密码进入系统，解决首次使用和换电脑后继续使用的问题。

**界面元素**

- 注册页：账号输入框、密码输入框、确认密码输入框、注册按钮、去登录入口
- 登录页：账号输入框、密码输入框、登录按钮、去注册入口

**用户流程**

```text
[注册入口]
  ↓
用户点击“注册”
  ↓
系统打开注册页
  ↓
用户输入账号、密码、确认密码并点击“提交注册”
  ├─ 成功 → 系统创建账号 → 用户看到“注册成功，请登录”并跳转登录页
  ├─ 失败（必填未填）→ 系统不提交 → 用户看到对应输入框下方提示
  ├─ 失败（两次密码不一致）→ 系统不提交 → 用户看到“两个密码不一致”
  └─ 失败（账号已存在）→ 系统不提交 → 用户看到“该账号已存在，请更换账号”

[登录入口]
  ↓
用户点击“登录”
  ↓
系统打开登录页
  ↓
用户输入账号和密码并点击“登录”
  ├─ 成功 → 系统验证账号信息并进入系统 → 用户看到总览页
  ├─ 失败（账号或密码错误）→ 系统拒绝登录 → 用户看到“账号或密码错误”
  ├─ 失败（必填未填）→ 系统不提交 → 用户看到“请填写完整后再登录”
  └─ 失败（网络异常）→ 系统中止本次操作 → 用户看到“网络异常，请稍后重试”
```

**交互逻辑**

- 用户点击“注册” → 系统检查是否填完整、两次密码是否一致 → 用户看到成功提示或错误提示
- 用户点击“登录” → 系统检查账号和密码 → 用户看到总览页或失败提示

**数据变化**

- 注册成功后，新增一个可登录账号
- 登录成功后，系统记住当前登录状态，后续页面按该账号展示对应记录

**状态机**

| 状态名称 | 进入条件 | 退出条件 | 用户可见表现 |
|----------|----------|----------|--------------|
| 初始状态 | 用户打开注册页或登录页 | 用户开始输入 | 输入框为空，按钮可见 |
| 填写中 | 用户开始输入信息 | 用户点击提交或清空内容 | 输入内容实时显示 |
| 提交中 | 用户点击注册或登录 | 成功或失败返回 | 按钮显示处理中，不重复点击 |
| 成功状态 | 注册成功或登录成功 | 跳转完成 | 显示成功提示或直接进入总览 |
| 失败状态 | 提交失败 | 用户修改内容后再次提交 | 显示错误提示 |

**字段规范**

| 字段名 | 类型 | 是否必填 | 长度限制 | 校验规则 | 错误提示 |
|--------|------|----------|----------|----------|----------|
| 账号 | 文本 | 是 | 4-30 个字 | 不能为空，不能与已有账号重复 | 请输入账号 / 该账号已存在 |
| 密码 | 文本 | 是 | 6-30 个字 | 不能为空 | 请输入密码 |
| 确认密码 | 文本 | 是 | 6-30 个字 | 必须与密码一致 | 两个密码不一致 |

**文案规范**

| 场景 | 文案内容 |
|------|----------|
| 空状态 | 本功能无空状态文案 |
| 加载中 | 正在提交，请稍候 |
| 操作成功 | 注册成功，请登录 / 登录成功 |
| 操作失败 | 提交失败，请检查后重试 |
| 权限不足 | 请先登录后继续操作 |
| 网络异常 | 网络异常，请稍后重试 |

**异常情况处理**

| 异常场景 | 触发条件 | 处理方式 | 用户反馈 |
|----------|----------|----------|----------|
| 必填未填 | 账号或密码为空 | 不提交，定位到对应输入框 | 请输入完整信息 |
| 两次密码不一致 | 注册时两次密码不同 | 不提交 | 两个密码不一致 |
| 账号重复 | 注册账号已存在 | 不提交 | 该账号已存在，请更换账号 |
| 登录失败 | 账号或密码错误 | 停留当前页 | 账号或密码错误 |
| 网络异常 | 提交过程失败 | 中止本次提交 | 网络异常，请稍后重试 |

### 功能 2：总览快捷入口

**功能描述**

用户可在总览页快速查看今天或最近的进货、出货、报价记录，并通过快捷按钮直接进入新建流程。

**界面元素**

- 快捷按钮：新建进货单、新建出货单、新建报价单
- 最近记录区：最近进货、最近出货、最近报价列表
- 顶部欢迎区

**用户流程**

```text
[进入总览]
  ↓
用户登录成功后进入总览页
  ↓
系统展示最近记录和快捷按钮
  ↓
用户点击“新建进货单 / 新建出货单 / 新建报价单”
  ├─ 成功 → 系统打开对应新建页面 → 用户看到对应表单
  ├─ 失败（登录失效）→ 系统拦截访问 → 用户看到“登录已失效，请重新登录”
  └─ 失败（页面加载异常）→ 系统无法打开目标页 → 用户看到“页面打开失败，请重试”

[查看最近记录]
  ↓
用户点击某条最近记录
  ├─ 成功 → 系统打开对应详情页 → 用户看到完整单据内容
  └─ 失败（记录不存在）→ 系统返回总览 → 用户看到“这条记录已不存在”
```

**交互逻辑**

- 用户点击快捷按钮 → 系统打开对应新建页面 → 用户看到可填写表单
- 用户点击最近记录 → 系统打开对应详情页 → 用户看到完整单据内容

**数据变化**

- 本功能本身不新增业务数据
- 用户通过快捷入口进入后，后续保存单据时会新增对应记录

**状态机**

| 状态名称 | 进入条件 | 退出条件 | 用户可见表现 |
|----------|----------|----------|--------------|
| 空状态 | 当前账号还没有业务记录 | 用户新建第一条记录 | 显示引导文案和快捷按钮 |
| 正常展示 | 有最近记录 | 用户点击查看或新建 | 展示最近记录列表 |
| 加载中 | 刚进入总览页 | 加载完成或失败 | 显示“正在加载总览信息” |
| 异常状态 | 总览加载失败 | 用户刷新后重试 | 显示错误提示 |

**字段规范**

本功能无用户输入字段。

**文案规范**

| 场景 | 文案内容 |
|------|----------|
| 空状态 | 还没有业务记录，先新建第一张单据 |
| 加载中 | 正在加载总览信息 |
| 操作成功 | 已打开对应页面 |
| 操作失败 | 页面打开失败，请重试 |
| 权限不足 | 登录已失效，请重新登录 |
| 网络异常 | 网络异常，请刷新后重试 |

**异常情况处理**

| 异常场景 | 触发条件 | 处理方式 | 用户反馈 |
|----------|----------|----------|----------|
| 无记录 | 当前账号没有任何单据 | 展示空状态和快捷入口 | 还没有业务记录，先新建第一张单据 |
| 登录失效 | 打开快捷入口前登录状态失效 | 跳转登录页 | 登录已失效，请重新登录 |
| 页面打开失败 | 目标页面未正常打开 | 保持在总览页 | 页面打开失败，请重试 |
| 网络异常 | 总览数据加载失败 | 支持刷新重试 | 网络异常，请刷新后重试 |

### 功能 3：货品管理

**功能描述**

用户可在货品页新增、编辑、查看、停用、删除货品，并查看当前库存数量，为后续进货、出货、报价提供基础信息。

**界面元素**

- 搜索框
- 新建货品按钮
- 货品列表
- 操作按钮：编辑、停用、启用、删除
- 货品表单：名称、规格、单位、默认进货价、默认销售价、图片、备注

**用户流程**


### 功能 5：进货单管理

**功能描述**

用户可创建、编辑、查看和导出进货单，一张单支持多个货品明细行，保存后会增加对应货品库存。

**界面元素**

- 进货列表
- 新建进货单按钮
- 进货表单：编号、日期、供应商、备注
- 明细区：货品、规格、单位、数量、单价、行金额、增加一行、删除一行
- 合计区
- 按钮：保存、导出打印版、导出表格版

**用户流程**

```text
[新建进货单]
  ↓
用户点击“新建进货单”
  ↓
系统打开进货单页面，并按规则生成编号
  ↓
用户选择日期、供应商，添加一个或多个货品明细，填写数量和单价后点击“保存”
  ├─ 成功 → 系统保存进货单并增加对应库存 → 用户看到“进货单已保存”
  ├─ 失败（必填未填）→ 系统不保存 → 用户看到“请填写完整后再保存”
  ├─ 失败（明细为空）→ 系统不保存 → 用户看到“请至少添加一条货品明细”
  ├─ 失败（编号冲突）→ 系统重新检查编号 → 用户看到“编号生成失败，请重试”
  └─ 失败（网络异常）→ 系统不保存 → 用户看到“网络异常，请稍后重试”

[编辑进货单]
  ↓
用户进入进货单详情页并点击“编辑”
  ↓
系统展示原内容
  ↓
用户修改明细或备注后点击“保存”
  ├─ 成功 → 系统更新单据并同步调整库存 → 用户看到“进货单已更新”
  └─ 失败 → 系统保留原内容不变 → 用户看到对应失败提示

[导出进货单]
  ↓
用户打开已保存的进货单详情页
  ↓
用户点击“导出打印版”或“导出表格版”
  ├─ 成功 → 系统按既有模板导出 → 用户看到“导出成功”
  └─ 失败 → 系统不影响已保存单据 → 用户看到“导出失败，请重试”
```

**交互逻辑**

- 用户点击“新建进货单” → 系统自动带出进货单编号 → 用户看到可填写表单
- 用户点击“添加一行” → 系统新增一行空白明细 → 用户看到新的货品行
- 用户选择货品 → 系统带出规格、单位和默认进货价 → 用户看到自动填入内容
- 用户填写数量和单价 → 系统计算每行金额和整单合计 → 用户看到实时更新的金额
- 用户点击“保存” → 系统保存单据并增加库存 → 用户看到成功提示
- 用户点击“导出” → 系统按既有模板整理单据 → 用户看到下载结果

**数据变化**

- 保存成功后，新增一张进货单
- 保存成功后，对应货品库存增加
- 编辑已保存进货单后，库存按修改后的差额同步调整
- 导出不会改变业务数据

**状态机**

| 状态名称 | 进入条件 | 退出条件 | 用户可见表现 |
|----------|----------|----------|--------------|
| 空状态 | 没有任何进货单 | 新建成功 | 显示“还没有进货单，先新建一张” |
| 填写中 | 打开新建或编辑页 | 点击保存或返回 | 表单可编辑，金额实时变化 |
| 保存中 | 点击保存 | 成功或失败返回 | 按钮显示处理中 |
| 保存成功 | 单据保存完成 | 关闭提示或继续导出 | 显示成功提示 |
| 保存失败 | 单据保存失败 | 用户修改后重试 | 显示失败原因 |

### 功能 6：出货单管理

**功能描述**

用户可创建、编辑、查看和导出出货单，一张单支持多个货品明细行，保存后会减少对应货品库存；当出货数量大于当前库存时允许保存，但必须给出明显提醒。

**界面元素**

- 出货列表
- 新建出货单按钮
- 出货表单：编号、日期、客户、备注
- 明细区：货品、规格、单位、当前库存、数量、单价、行金额
- 合计区
- 按钮：保存、导出打印版、导出表格版

**用户流程**

```text
[新建出货单]
  ↓
用户点击“新建出货单”
  ↓
系统打开出货单页面，并按规则生成编号
  ↓

  ├─ 成功（库存不足）→ 系统弹出明显提醒，用户确认后保存 → 用户看到“出货单已保存，部分货品库存不足”
  ├─ 失败（必填未填）→ 系统不保存 → 用户看到“请填写完整后再保存”
  ├─ 失败（明细为空）→ 系统不保存 → 用户看到“请至少添加一条货品明细”
  └─ 失败（网络异常）→ 系统不保存 → 用户看到“网络异常，请稍后重试”

[编辑出货单]
  ↓
用户点击已有出货单进入详情页
  ↓
用户点击“编辑”并修改内容
  ├─ 成功 → 系统更新单据并同步调整库存 → 用户看到“出货单已更新”
  └─ 失败 → 系统不更新 → 用户看到失败原因

[导出出货单]
  ↓

  ├─ 成功 → 系统按既有模板导出 → 用户看到“导出成功”
  └─ 失败 → 系统保留原单据不变 → 用户看到“导出失败，请重试”
```

- 用户填写数量 → 系统比对当前库存 → 用户看到库存信息和是否不足提示
- 用户点击“保存” → 系统减少库存 → 用户看到保存结果
- 用户点击“导出” → 系统按既有模板整理出货单 → 用户看到导出结果

- 存储使用量、容量状态和上次成功备份时间
- “导出加密备份”“恢复备份”按钮
- 新版本提示、更新说明和“备份后更新”入口

**备份流程**

```text
用户点击导出备份
-> 系统统计全部账号、业务数据和附件
-> 用户输入并确认独立备份密码
-> 系统校验、序列化、压缩并使用 AES-256-GCM 加密
-> 下载 .tradgio-backup 文件
-> 记录本次成功备份时间和数据摘要
```

备份密码不保存，忘记后无法恢复。备份包含全部账号及密码校验数据，但不包含活动 session。

**恢复流程**

```text
用户选择 .tradgio-backup
-> 输入备份密码
-> 系统解密、解压并校验格式、摘要、版本和容量
-> 展示账号数、记录数、附件数、备份时间和替换影响
-> 用户确认整机替换
-> 系统自动生成恢复前快照
-> 使用 IndexedDB transaction 替换数据
-> 重新打开应用并要求登录
```

密码错误、文件篡改、版本不兼容、容量不足或写入失败时不得修改当前数据。

**存储与附件规则**

- 合同附件以 Blob 保存，默认单文件上限 20 MB。
- 存储使用率达到 70% 时持续提醒备份和清理。
- 达到 85% 时阻止继续写入附件和其他大体积数据。
- 正式使用限定 Windows 最新稳定版 Chrome 或 Edge，不支持无痕模式作为正式环境。

**版本更新规则**

- Mac 开发发布后，Windows PWA 检测新版本并显示提示。
- 不强制刷新正在填写的页面。
- schema 变化或高风险更新必须先提示生成备份。
- 更新失败继续运行上一稳定版本，不删除现有 IndexedDB。
- 正式 HTTPS Origin 投入使用后不得随意更换；更换时必须通过备份与恢复迁移数据。

**本地失败模型说明**

本 PRD 早期功能章节中的“网络异常”是原平台化假设下的通用失败文案。采用 ADR-0002 后，核心 CRUD、搜索、合同和导出在本地运行，对应实现应优先映射为“本地数据读取失败”“本地保存失败”“存储空间不足”“模板缓存不可用”或“应用版本不兼容”；只有首次加载和版本更新仍属于网络异常。
