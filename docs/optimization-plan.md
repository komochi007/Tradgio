# Tradgio 本轮优化计划

日期：2026-06-03

本文档记录本轮视觉、交互、数据看板、草稿和模板导出的已确认需求。后续开发按本文档拆分任务推进，避免把跨模块能力散落在页面层。

## 1. 已确认优化范围

### 1.1 总览页

- 移除总览页顶部 `Tradgio / 库存管理平台` 介绍卡片。
- 移除总览页底部独立跨模块搜索卡片。
- 欢迎语、日期、天气信息放入总览页页头左侧。
- 天气图标重新设计，避免使用当前 emoji 样式。
- 跨模块搜索放入总览页页头右侧。
- 搜索范围固定为：进货单、出货单、报价单、合同。
- 搜索结果以下拉框展示，点击结果跳转对应详情页。
- 总览页新增 3 张数据看板卡片：
  - 环形图：按产品类型展示产品 SKU 数量。
  - 柱状图：按月份展示产品出货数量，右上角支持产品类型筛选。
  - 柱状图：按月份展示产品出货金额，右上角支持产品类型筛选。
- 看板默认统计近 6 个月，月份无数据补 0，产品类型为空归为“未分类”。
- 图表支持悬浮提示。

### 1.2 视觉系统

- 产品 Logo 改为 Tradgio 品牌字母标。
- 左侧导航图标重新设计，统一为更简洁、高级的品牌化图标语言。
- 总览页新建进货单、新建出货单、新建报价单入口图标重新设计。
- 所有业务页移除顶部重复小标题，仅保留内容区主标题和说明。
- 业务界面保持桌面端优先、冷静、专业、高频录单友好，不做营销化视觉。

### 1.3 表单与通用交互

- 所有业务列表搜索栏加宽，确保 placeholder 完整显示。
- 单据新建/编辑页中，货品明细标题与表格之间增加间距。
- 原生 `select` 替换为统一自定义下拉组件。
- 同类下拉框统一排查，包含普通表单下拉、筛选下拉和导出相关下拉。
- 修复单据列表中导出下拉菜单被表格容器遮挡的问题。

### 1.4 草稿能力

- 草稿覆盖所有新建页：
  - 新建货品
  - 新建往来单位
  - 新建进货单
  - 新建出货单
  - 新建报价单
  - 新建合同
- 草稿保存方式为自动保存 + 手动保存。
- 进入新建页时如存在草稿，展示顶部提示条，用户可选择恢复或丢弃。
- 正式创建成功后清除对应草稿。
- 首版不覆盖编辑页，不做多草稿列表。
- 草稿按账号 + 表单类型隔离，当前本地 MVP 使用 localStorage 实现。

### 1.5 模板导出

- 出货单、报价单导出菜单保留“打印版”。
- 原“表格版”改为“模板Excel”。
- 模板导出首版使用 `.xlsx`。
- 当前用户提供的 `.xls` 模板已转换为 `.xlsx` 并放入标准目录：
  - `public/templates/sales-order-template.xlsx`
  - `public/templates/quote-template.xlsx`
- 原始 `.xls` 文件暂保留，作为用户提供的原件备份。
- 模板字段缺失时，优先补到对应出货单/报价单表单并保存，不在导出时临时拼凑。
- 模板字段映射和文件渲染必须收敛在 `Export Service`，页面层只负责触发导出和显示结果。
