# 上线后版本与迭代管理

本文档定义 Tradgio 上线后的功能优化、缺陷修复、版本发布和回滚管理规则。上线前生产化路线仍以 `tasks/production-roadmap.md` 和相关验收报告为历史基线；上线后的日常迭代从 `tasks/post-launch-backlog.md` 开始管理。

## 1. 管理原则

- 采用“仓库 Markdown + 小步补丁 + 稳妥门禁”的轻量流程。
- 当前仍保持单人业务系统定位，不引入多人协作、复杂权限或重型项目管理工具。
- GitHub Issues 暂不作为主入口；如后续多人参与，再把 backlog 条目迁移或同步到 Issues。
- 日常小修不再追加到 `tasks/production-roadmap.md`，该文件作为上线前历史依据保留。
- 涉及 IndexedDB schema、备份恢复、账号隔离、库存、导出模板、PWA 更新的变更，自动按高风险发布处理。

## 2. 文档入口

| 文档 | 用途 |
|---|---|
| `tasks/post-launch-backlog.md` | 记录上线后的 Bug、优化、新功能和运维事项 |
| `tasks/release-checklist.md` | 每次发布前后的日常检查清单 |
| `tasks/release-log.md` | 记录每个正式发布版本、提交号、验证结果和回滚点 |
| `docs/deployment.md` | 记录 GitHub Pages、PWA、缓存、更新和回滚细节 |

## 3. 任务分级

| 优先级 | 定义 | 默认处理 |
|---|---|---|
| `P0` | 数据丢失、无法登录、核心录单不可用、备份恢复不可用、安全问题 | 立即热修 |
| `P1` | 核心业务错误但有绕行办法 | 进入最近补丁版 |
| `P2` | 明确提升效率的小功能或体验优化 | 排入后续补丁版或小版本 |
| `P3` | 非紧急优化、想法池、低频需求 | 暂缓评估 |

## 4. 任务状态

任务状态统一使用：

- `待澄清`：目标、影响范围或验收标准不明确。
- `待排期`：信息完整，但尚未进入当前版本。
- `进行中`：已经开始设计、实现或验证。
- `待验收`：实现完成，等待用户确认或发布前验证。
- `已发布`：已进入正式地址并完成发布后最小验证。
- `暂缓`：暂时不做，但保留记录。
- `关闭`：确认无需处理、重复或已被其他任务覆盖。

## 5. 版本规则

- 当前正式版基线为 `0.1.0`。
- `0.1.x` 用于 Bug 修复、小体验优化和无数据结构变化的补丁。
- `0.2.0+` 用于用户可感知的新功能组合。
- 涉及 IndexedDB schema 升级时，必须记录 schema 版本、迁移方案、升级前备份和回滚限制。
- 低版本回滚如果无法兼容新 schema，不能直接发布旧版本，应发布兼容修复版。

## 6. Backlog 准入要求

每个需求进入开发前必须补齐：

- 问题或目标
- 影响用户
- 所属模块
- 优先级
- 风险等级
- 验收标准
- 验证命令

修改超过 3 个文件时，必须先拆成小步骤；每一步说明目标、影响范围和验证方式。

## 7. 发布流程

1. 把用户反馈或新想法登记到 `tasks/post-launch-backlog.md`。
2. 先做分级和范围判断，不明确的需求保持 `待澄清`。
3. 每次只从 backlog 中挑选少量高确定性事项进入当前补丁版。
4. 开发完成后按风险等级执行验证。
5. 发布前更新 `tasks/release-log.md` 和 `tasks/release-checklist.md`。
6. 手动触发现有 `Deploy PWA` 工作流发布。
7. 发布后做最小线上验证：打开应用、登录、查看核心数据、进入备份页、刷新后确认正常。

## 8. 验证门禁

| 变更类型 | 最低验证 |
|---|---|
| 文案、样式小修 | `npm run quality` |
| 表单、列表、路由、核心交互 | `npm run quality`，必要时手动浏览器验证 |
| 录单、库存、搜索、导出 | `npm run quality` + `npm run test:e2e` |
| PWA、发布、缓存、更新、回滚 | `npm run quality` + `npm run test:e2e` + `npm run test:pwa` |
| schema、备份恢复 | 自动化测试、真实浏览器恢复演练、发布前加密备份、发布后数据核对 |

## 9. 高风险发布规则

以下变更必须按高风险发布处理：

- 库存计算、库存流水或库存快照
- 账号隔离、登录认证或密码迁移
- IndexedDB schema、Repository、迁移器
- 合同附件 Blob、备份恢复、容量策略
- 导出模板、Export Service、固定模板缓存
- Service Worker、PWA 更新、发布和回滚策略

高风险发布必须满足：

- 发布前导出完整加密 `.tradgio-backup`，并确认至少两处副本。
- 在 `tasks/release-log.md` 记录回滚提交号。
- schema 升级版本必须做真实升级验收。
- 发布后执行数据核对和最小线上验证。
